Switch to multi-user mode

This server is currently in single-user mode: every caller is implicitly authenticated as the owner account with no token required. Enable multi-user mode to require Bearer-token auth and add per-user isolation.

How the transition works:

  1. A one-time bootstrap token has been written to state/admin_bootstrap.txt on the server filesystem.
  2. Read it from disk and paste it below along with the username and password you want for the first admin account.
  3. On submit, all data currently owned by owner moves to your new admin account, the sentinel is removed, and the bootstrap file is deleted. Subsequent requests require auth.

The bootstrap file is intentionally not displayed in the browser. Reading it requires filesystem access to the server (or its bind mount), which is the trust boundary for granting admin.
