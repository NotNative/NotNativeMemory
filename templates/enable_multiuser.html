This server is currently in single-user mode: every caller is implicitly authenticated as owner with no token required. Claiming the instance creates the root principal, issues a root Bearer token, adopts legacy memories into root ownership, and requires tokens for future access.

How the transition works:

  1. A one-time bootstrap token has been written to state/admin_bootstrap.txt on the server filesystem.
  2. Read it from disk and paste it below.
  3. On submit, all data currently owned by owner moves to root, the sentinel is removed, and the bootstrap file is deleted.
  4. The root token is shown once for root NNA configuration.

The bootstrap file is intentionally not displayed in the browser. Reading it requires filesystem access to the server (or its bind mount), which is the trust boundary for claiming root.
